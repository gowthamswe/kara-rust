  - [x] **Round 12.6 (2026-05-02) — repeatable-closure dispatch.** Pinned existing detection infrastructure with explicit tests: closures with `ref |...|` / `mut ref |...|` prefixes are not flagged once-callable and dispatch correctly across multiple invocations through the interpreter. Detection pre-existed in `src/ownership.rs` (`once_callable_closures: HashSet` is populated only when `MoveKind::ClosureCapture` retag fires for a captured owned non-Copy outer value; the K2 enforcement path errors-and-skips-retag for explicit-prefix closures that consume a capture, and never retags at all when the body only reads). 5 ownership tests + 3 interpreter tests covering `ref ||` multi-call, `mut ref ||` multi-call, outer-binding-still-usable-after-call, bare-form regression, in-loop dispatch, and `Vec[Fn() -> i64]` slot dispatch via `vec[i]()`. Commit `895d7bb`.
  - [x] **Once-callable closure rejected when invoked through a `ref` binding.** A `ref Fn(...)` parameter or loop variable yielding `ref Fn(...)` whose underlying closure is once-callable should reject the invocation with a clear error. The legacy `once_callable_closures: HashSet<String>` (ownership.rs:317) was keyed by binding name in a single function scope and populated only at let-RHS-is-closure sites (ownership.rs:880-893); lifting once-callability to a property that propagates through types and across function / container boundaries was a 5-step build. Steps 1–5 closed in rounds 12.43–12.47. The diagnostic now names the consumed capture, the slot type, and offers three concrete fixes; integration tests cover parameter passing, method receiver, struct field assignment, generic-function instantiation, and loop invocation end-to-end (`tests/closures.rs`). Sub-rounds below in dependency order.

    - [x] **Round 12.43 (2026-05-04) — Step 1: Type-system carrier for once-callability.** Added `Type::OnceFunction { params, return_type }` as a sibling variant to `Type::Function` in `src/typechecker.rs`. `type_display` now prints `OnceFn(...)` and `OnceFn(...) -> T`. `types_compatible` accepts the `(OnceFunction, OnceFunction)` identity pair and rejects every cross-pair (`Function` ⇄ `OnceFunction` falls through to `_ => false`) — the conservative subtyping baseline for Step 1; later steps may relax `Function ⇒ OnceFunction` widening if needed at parameter slots. Numeric-comparability "false" arms in the six `type_supports_*` queries (`partial_eq` / `eq` / `hash` / `ord` / `display` / `partial_ord`) widened to include `OnceFunction { .. }`. `contains_type_param` / `substitute_type_params` / `resolve_assoc_projections` propagate once-ness through generic substitution. Call-site dispatch in `Call` matches `Type::Function | Type::OnceFunction` so an owned `OnceFn` callee types normally (the actual rejection at slot boundaries lives in Step 3). `src/ownership.rs:2627` closure-param-type-inference path recognizes the new variant via or-pattern. Closure literals still synthesize as `Type::Function` (Step 2 promotes the capture-consume case to `OnceFunction`); `lower_type_expr` for `TypeKind::FnType` still produces `Function` (no surface `OnceFn(...)` annotation yet — defer until Step 2 lands so inference and surface form stay aligned). 9 new unit tests in `once_function_carrier_tests` (src/typechecker.rs) cover display round-trip (default + unit-return + multi-param), identity, cross-pair rejection (both directions), arity mismatch, all six numeric-trait "false" arms, `substitute_type_params` once-ness preservation, and `contains_type_param` traversal.
    - [x] **Round 12.44 (2026-05-04) — Step 2: Closure-expression once-callability inference at construction.** The typechecker now assigns `Type::OnceFunction` to a closure expression iff its body consumes a captured outer non-Copy binding; otherwise `Type::Function`. Detection lives entirely inside `src/typechecker.rs` (no cross-phase rewrite of `expr_types`). New helpers on `TypeChecker`: `flatten_local_scope_snapshot` captures outer-name → type just BEFORE the closure pushes its param scope so closure params and body-local lets are excluded from the capture set; `is_copy_type_during_check` mirrors `ownership::is_copy_type` against the in-progress `env.structs` / `env.enums` / `env.distinct_types`; `closure_consumes_captured_non_copy` runs a Reading/Consuming-mode body walk (`walk_capture_consume` + `walk_capture_consume_block`) mirroring `use_classifier::walk_expr` semantics — owned-arg `Call` slots (decided by `callee_borrow_positions` reading `env.functions`/`env.impls` directly), `StructLiteral` fields, `Return(Some)` / `Break(Some)` / `Question` / `Match` scrutinee with binding arms, and `For` iterables flip into Consuming; `MethodCall` receiver stays Reading (conservative — the typechecker doesn't track `SelfParam` modes, leaving owned-self method-receiver consumes for Step 3 if needed). Shadow-stack tracking handles closure params, body lets, `if let` / `while let` / `for` patterns, match-arm bindings, and nested closures so a body identifier matching a local-scope name isn't mistaken for a capture. Both Closure arms (`infer_expr` synth + `check_expr` expected-type pushdown) call `closure_type_with_capture_inference`, which short-circuits to `Type::Function` for explicit `ref ||` / `mut ref ||` prefixes (matches round 12.6's repeatable-closure rule). New `CaptureWalkMode` enum at module scope. 8 new unit tests in `closure_once_callability_inference_tests` (src/typechecker.rs) covering captures-and-consumes → `OnceFunction`, only-reads-capture → `Function`, capture-free → `Function`, explicit `ref ||` and `mut ref ||` force `Function`, Copy capture → `Function`, closure param shadows outer non-Copy capture → `Function`, body-local `let` shadows outer non-Copy capture → `Function`. Existing test suite (lib + integration + LLVM codegen) green; only pre-existing `pure`-keyword failure remains.
    - [x] **Round 12.45 (2026-05-04) — Step 3: Reject once-callable in `ref Fn(...)` and `Fn(...)` parameter slots (caller-side).** New diagnostic `OnceFnIntoFnSlot` (E0235) fires whenever a once-callable closure value flows into a `Fn`-shaped slot. Detection lives in `check_assignable` (src/typechecker.rs): the new helper `is_once_into_fn_shape` strips ref / mut-ref wrapping from BOTH expected and found, then matches `(Function, OnceFunction)` — so bare-bare, ref-ref, and cross-wrapped (e.g. bare `OnceFn` arg into `ref Fn` slot) all unify to the same once-callability violation rather than degrading into a generic ref-mismatch. The diagnostic message names the slot type, the actual closure type, and (when the rejected expression's span keys into the new `closure_once_reasons` side-table) the consumed captured binding. Step 2's walker now records a `OnceReason { consumed_binding, consumed_span }` instead of a bare bool; `closure_type_with_capture_inference` populates `closure_once_reasons` keyed by the closure-expression span on every once-callable inference. Because every assignment boundary routes through `check_assignable`, the new error fires at function-call args (own- and ref-Fn slots), method-call args, let bindings, function returns, and field-init slots without any per-site plumbing. The TypeMismatch double-report is suppressed for this exact shape; a span-equality test pins the single-error invariant. 7 new unit tests in `once_fn_slot_rejection_tests` (src/typechecker.rs) cover own-Fn slot rejection (with consumed-capture name in the message), own-Fn slot accepting a repeatable closure, own-Fn slot accepting a let-bound `ref ||` closure (parser rejects raw `ref` at call-arg position so the test goes through a binding), ref-Fn slot rejection (cross-wrapped via `unwrap` recursion), method-call Fn-slot rejection, cross-call forwarding rejecting at the outer site, and the no-double-report invariant. New `TypeErrorKind::OnceFnIntoFnSlot` variant + `E0235` mapping in `cli.rs` rendering. Test suite green except the pre-existing `pure`-keyword failure documented in 12.43/12.44.
    - [x] **Round 12.46 (2026-05-04) — Step 4: Reject once-callable in container element slots; loop-variable invocation falls out for free.** Added a narrow `Vec.push` slot dispatch in `infer_method_call` (src/typechecker.rs) that fires *before* the silent fall-through Vec used to take: when the receiver is `Vec[T]` (or `ref` / `mut ref` over one) and method is `push` with one arg, the argument routes through `check_assignable(element, arg_ty, span)` so Step 3's `OnceFnIntoFnSlot` (E0235) fires automatically when an inferred `OnceFunction` value flows into a `Vec[Fn(...)]` element slot. Other Vec methods (`pop`, `len`, `first`, …) continue through the historical permissive fall-through to avoid surfacing untyped slot regressions outside Step 4's scope; Step 5 can promote them. Surface `OnceFn(...)` annotation now parses (parser branch was `Token::Identifier(s) if s == "Fn"` → `s == "Fn" || s == "OnceFn"`); a new `is_once: bool` field on `TypeKind::FnType` (src/ast.rs) flows the parser flag through to `lower_type_expr`, which emits `Type::OnceFunction` when set. Resolver / effectchecker / formatter destructurings of `FnType` updated to absorb the new field (effectchecker writes `OnceFn(` instead of `Fn(` in trace formatting when `is_once` is set; resolver and formatter ignore it). Both `Vec[Fn(...)]` and `Vec[OnceFn(...)]` annotations now lower correctly. For-loop iteration falls out for free: `for f in vec` over `Vec[Fn]` yields `f: Function` and the body's `f()` dispatches through Step 1's `Function | OnceFunction` Call match; same path works for `Vec[OnceFn]` iteration where each iteration owns its `OnceFn` and the single body invocation succeeds. 7 new unit tests in `once_fn_container_slot_tests` (src/typechecker.rs) cover Vec[Fn] push of an OnceFn closure literal rejecting (with consumed-capture name in message), Vec[Fn] push of a repeatable closure accepting, Vec[OnceFn] push of an OnceFn closure accepting, surface `OnceFn(...)` annotation lowering correctness via the reverse-direction TypeMismatch (Function-into-OnceFn slot rejects with `TypeMismatch` not `OnceFnIntoFnSlot` — pinning that the annotation does not silently widen to Function), Vec[Fn] iteration with `f()` body, Vec[OnceFn] iteration with `f()` body, and Vec[Fn] push of a let-bound once-callable closure (pinning that the slot check applies to any value, not just literals). Test suite green except the pre-existing `pure`-keyword failure documented in 12.43–12.45.
    - [x] **Round 12.47 (2026-05-04) — Step 5: Diagnostic polish + integration sweep.** `OnceFnIntoFnSlot` (E0235) message construction in `check_assignable` (src/typechecker.rs) now appends a `; help: ...` suffix listing the three concrete fixes — clone the consumed capture so the closure becomes repeatable, restructure to invoke the closure locally instead of routing it through this slot, or change the slot type to `OnceFn(...)` if the user controls its declaration. The fix-hint suffix is unconditional (independent of whether the closure-once-reasons side-table found a consumed binding) so it fires for both literal-rejection and forwarded-value-rejection sites. New `tests/closures.rs` integration file (14 tests) crosses the five propagation paths end-to-end through `parse → resolve → typecheck`: parameter passing (own `Fn` and `ref Fn`), method receiver, struct field assignment in struct-literal init (incl. reverse `OnceFn` field accepting a once-callable literal), generic-function instantiation under `[T]` substitution (Fn-vs-OnceFn dimension is independent of T), and loop invocation over `Vec[Fn]` / `Vec[OnceFn]` (push-site rejection vs body-invocation success). Each path includes both rejection and acceptance cases; the diagnostic-polish test asserts all three fix-hint phrases appear in the message. New unit test `diagnostic_includes_three_concrete_fix_hints` in `once_fn_slot_rejection_tests` (src/typechecker.rs) pins the same contract at the unit level. Regression guards green: `tests/ownership.rs` 154/154, `tests/rc_predicate_parity.rs` 35/35, full non-codegen suite + LLVM-gated suite (with `KARAC_SKIP_ASAN_TESTS=1` for the platform-specific LeakSanitizer-on-macOS skip) 100% pass; clippy + rustfmt clean.
  - [x] **Round 12.48 (2026-05-04) — Interpreter `mut ref |...|` capture semantics — mutations propagate back across calls.** Mutations made by a `mut ref` closure body to a captured outer binding now persist across invocations and are visible at the outer binding after the calls return. New `Value::SharedCell(Arc<Mutex<Value>>)` variant aliases an outer-scope slot with the same cell stored in the closure's `closure_env` map. `Env::get` auto-derefs the cell so every reader observes the underlying value; `Env::set` checks for `SharedCell` in the matched slot and writes through the cell so reads on either side see the mutation. New `Env::wrap_capture` promotes a binding's slot in-place at closure construction. A targeted free-variable walker (`collect_free_idents_expr` / `collect_free_idents_block`, mirroring `Pattern::binding_names` for shadow tracking) identifies which outer names the closure body uses; only those slots get promoted, and globals living in `scopes[0]` (functions, enum variants, type ctors) are skipped. `Arc<Mutex<...>>` rather than `Rc<RefCell<...>>` so a `par {}` block can clone branch envs across thread boundaries — the typechecker already rejects sharing a mutable capture across `par` branches, so single-threaded mutation in practice. Bare and `ref ||` closures stay snapshot-and-clone (the existing `test_ref_closure_*` tests pin this; `test_bare_closure_does_not_propagate_mutation` pins the negative case for bare). 7 new interpreter integration tests in `tests/interpreter.rs` cover assignment propagation, compound-assign propagation, `Vec.push` propagation, struct-field mutation propagation, observe-outer-mutation-between-calls bidirectional aliasing, forwarding through a higher-order `ref Fn()` parameter, and the bare-closure negative case. Full test suite + LLVM-gated suite green; clippy + rustfmt clean. Closes the closure-calling-through-`ref` umbrella.
